## Control de performance y escalación

Definir target operativo inicial antes de cierre D1 (campos mínimos):

- `target_reply_rate_total`
- `target_booked_calls_rate`
- `target_neutralizacion_objeciones`

Regla de escalación inmediata a Board/CEO:

- Si performance cae `>20%` vs target operativo definido.
- Si el lanzamiento queda bloqueado `>24h`.

Formato de escalación:

- Motivo
- Métrica afectada vs target
- Impacto comercial esperado
- Acción correctiva propuesta
- Decisión requerida

## Goal Gate antes de propuesta

No enviar propuesta si falta alguno:

- `ICP fit` alto/medio
- `presupuesto` confirmado
- `autoridad` confirmada

Si el deal estimado es `>$50k`, abrir escalación y esperar sign-off de board antes de enviar propuesta.
